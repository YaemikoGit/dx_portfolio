Clear Care (Module: Account & Data Hub)

Project Brief

The ClearCare system is a modular healthcare management solution designed to facilitate patient care, appointment scheduling, and medication tracking. For this team’s project which is responsible for the module, which is also known as Account & Data Hub, which manages patient accounts, storing and retrieving medical history, and providing essential user details to other components in the system.

Additionally, it aims to facilitate interoperability with the other system components. Hence, learnt the fundamentals of software design on a bigger scale through this project.

Skill

Software Design, Web development

Challenges

To understand what are the user features in this module, what quality attributes do the team prioritize in this project and what are the business processes to focus on while designing the module. Additionally, while designing the module’s infrastructure architecture, what types or patterns should the team use in order to achieve the following quality attributes for this project.

Results

ErrandGo is able to function as intended following the core objectives. The Account & Data Hub Module is able to serve its purpose as the core data management module with a mix of layered architecture and Model-View-Controller pattern. This infrastructure is split into three layers namely: Presentation, Domain and Data layer. Moreover, the team has managed to lay out a component diagram which allows the team to have a better understanding of the whole system to structure the module accordingly.

Highlights

Key Features

In this module, the team has come up with a few main user features essential for the Clear Care system. This diagram depicts which users (Doctor and Nurse) will require which user features as shown in the oval shape.

User Profile Management:

Users can view and edit their profile details.

View & Export Medical Record:
  • Doctors and nurses can view medical records associated with their patients.
  • Doctors and nurses can also export medical records to any format they wish.
Create Patient Medical Record

Doctors can create a medical record for a patient, ensuring it remains permanently stored.

Filing Erratum to Update Medical Records

Doctors can file an erratum to update the original patient's medical record.


Architecture

The Account and Data Hub (Module 1) serves as the core data management module in the ClearCare system, responsible for user authentication, profile management, and secure medical record handling.

The system follows a layered architecture utilizing the MVC (Model-View-Controller) pattern for the presentation layer, a domain model for business logic, and a combination of data mapper and gateway patterns for the data layer to ensure modularity, maintainability, and security.

Business Process: Create Patient Medical Record

The implementation follows a Table Module Pattern, which maps directly to database tables, and a Table Data Gateway Pattern to handle database communication.

This diagram depicts a systematic flow of creating a Patient Medical Record with various tables, classes and the database used in this process.
