Errand Go

Project Brief

ErrandGo is an application that allows users to post errands on the platform itself, in which "runners" will take on any of the errands and complete them. This platform serves as a function for both the service providers (runners) and the users who require these errands to be completed. Main role: In-charge of key feature Account Management

Skill

Web development, Software Design

Challenges

Designing and developing a user-friendly and inclusive errand platform for key audiences such as working professionals, students and elderly to enhance overall user satisfaction through identifying common usability issues and strengths of existing platforms. Additionally, the team focuses on fostering user engagement through advanced features such as the chat system to enable direct communication between users and profile customization to tailor their personal preferences.

Results

ErrandGo is able to function as intended following the core objectives. Key considerations are status filters for tracking, color-coded status and differentiated chat message formatting for clarity and clear navigation, ensuring it fosters user engagement.

Architecture

The system follows a Model-View-Controller structure, with business logic separated into service layers.

Frontend:

Developed using HTML, CSS and JavaScript.

Backend:

Built using Python with the Flask web framework for modularity and maintainability, along with Jinja2 for server-side templating for dynamic rendering.

APIs Integrated:

Stripe Gateway to manage payment across the platforms.

Database:

Neon-hosted Postgres as the main database which interacts via SQLAlchemy as Object-Relational Mapper, abstracting the data written to its models with Python classes.
